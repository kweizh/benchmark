For example, the\r\nfollowing template is labeled as `op_log operation id short` automatically:\r\n\r\n```sh\r\njj op log -T 'self.id().short()'\r\n```\r\n\r\nThe exact names of such labels are often straightforward, but are not currently\r\ndocumented. You can discover the actual label names used with the\r\n`--color=debug` option, e.g.\r\n\r\n```sh\r\njj op log -T 'self.id().short()' --color=debug\r\n```\r\n\r\nAdditionally, you can **manually** insert arbitrary labels using the\r\n`label(label, content)` function. For example,\r\n\r\n```sh\r\njj op log -T '\"ID: \" ++ self.id().short().substr(0, 1) ++ label(\"id short\",  \"<redacted>\")'\r\n```\r\n\r\nwill print \"ID:\" in the default style, and the string `<redacted>` in the same\r\nstyle as the first character of the id. It would also be fine to use an\r\narbitrary template instead of the string `\"<redacted>\"`, possibly including\r\nnested invocations of `label()`.\r\n\r\nYou are free to use custom label names as well.
